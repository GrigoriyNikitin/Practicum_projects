# Платформа для блогеров «Блогикум»

## 1. Краткое описание проекта

Блогикум — социальная сеть для публикации личных дневников.
Это сайт, на котором пользователь может создать свою страницу и публиковать на ней сообщения («посты»). 
Для каждого поста нужно указать категорию — например «путешествия», «кулинария» или «python-разработка», а также опционально локацию, с которой связан пост, например «Остров отчаянья» или «Караганда». 
Пользователь может перейти на страницу любой категории и увидеть все посты, которые к ней относятся.
Пользователи смогут заходить на чужие страницы, читать и комментировать чужие посты.
Для своей страницы автор может задать имя и уникальный адрес.
Администратор сайта имеет возможность модерировать записи и блокировать пользователей, если начнут присылать спам.

## 2. Как запустить проект:

Клонировать репозиторий (проект в папке репозитория) и перейти в него в командной строке:

```
git clone --depth 1 --filter=blob:none --sparse https://github.com/GrigoriyNikitin/Practicum_projects.git
```

```
cd Practicum_projects
```

```
git sparse-checkout set blogicum/
```

```
cd blogicum/
```

Cоздать и активировать виртуальное окружение:

```
python -m venv venv
```

```
source venv/Scripts/activate
```

Установить зависимости из файла requirements.txt:

```
python -m pip install --upgrade pip
```

```
pip install -r requirements.txt
```

Выполнить миграции:

```
python blogicum/manage.py migrate
```

Запустить проект:

```
python blogicum/manage.py runserver
```

## 3. Примеры запросов:

### 3.1. Создание нового произведения
Производится POST-запрос к следующему эндпоинту:
```
/api/v1/titles/
```

Пример тела запроса к АПИ: 
```
{

    "name": "string",
    "year": 0,
    "description": "string",
    "genre": 

    [
        "string"
    ],
    "category": "string"

}
```

Пример ответа АПИ:

```
{

    "id": 0,
    "name": "string",
    "year": 0,
    "rating": 0,
    "description": "string",
    "genre": 

[

    {}

],
"category": 

    {
        "name": "string",
        "slug": "^-$"
    }

}
```

### 3.2. Создание нового отзыва
Производится POST-запрос к следующему эндпоинту:
```
/api/v1/titles/{title_id}/reviews/
```

Пример тела запроса к АПИ: 
```
{

    "text": "string",
    "score": 1

}
```

Пример ответа АПИ:

```
{

    "id": 0,
    "text": "string",
    "author": "string",
    "score": 1,
    "pub_date": "2019-08-24T14:15:22Z"

}
```
